## AIエージェント開発とは

AIエージェントを用いた開発とは、AI（人工知能）がコーディング、デバッグ、デプロイなどの開発タスクを支援または自動化する開発手法です。

### 主な特徴

- **対話型開発**: 自然言語でAIに指示を出し、コードを生成・修正
- **自動化**: 繰り返しタスクの自動実行
- **コンテキスト理解**: プロジェクト全体の構造を理解した上での提案
- **マルチタスク対応**: コーディングからデプロイまで一貫したサポート

---

## 使用したツール

### Claude Code

Anthropic社が提供する、コーディングに特化したAIアシスタント。

#### 主な機能

1. **ファイル操作**
   - ファイルの読み書き
   - コードの編集
   - ディレクトリ構造の理解

2. **コマンド実行**
   - Bashコマンドの実行
   - npm、git などのツールの操作
   - ビルド・デプロイの自動化

3. **コード理解**
   - プロジェクト構造の把握
   - 既存コードの解析
   - アーキテクチャの提案

4. **デバッグ支援**
   - エラーメッセージの解析
   - 問題の特定と修正提案
   - テストの実行とフィードバック

### その他のツール

- **Vite**: 高速な開発サーバーとビルドツール
- **TypeScript**: 型安全な開発環境
- **React**: UIライブラリ
- **Vercel**: デプロイプラットフォーム
- **GitHub**: バージョン管理とリポジトリホスティング

## 開発プロセス

### 1. プロジェクト初期化

```
ユーザー: "TypeScriptとReactでMarkdownビューアーを作って"
↓
AIエージェント:
- Viteプロジェクトの初期化
- 必要なパッケージのインストール
- 基本的なディレクトリ構造の作成
```

### 2. 機能実装

```
ユーザー: "dlディレクトリを作ってファイルをダウンロードするページを作れる？"
↓
AIエージェント:
- public/dlディレクトリの作成
- DownloadPageコンポーネントの実装
- ルーティングの設定
- サンプルファイルの配置
```

### 3. デプロイ

```
ユーザー: "Vercelにデプロイして"
↓
AIエージェント:
- Gitリポジトリの初期化
- Vercel CLIのセットアップ
- 認証プロセスのサポート
- 本番環境へのデプロイ
```

### 4. 継続的な改善

```
ユーザー: "リポジトリをpublicにして"
↓
AIエージェント:
- GitHub CLIのインストール
- 認証の設定
- リポジトリ設定の変更
- 変更のプッシュ
```

## 実践例：このサイトの構築

### Phase 1: プロジェクトのセットアップ

**実行時間**: 約5分

```bash
# AIが実行したコマンド
npm create vite@latest markdown-viewer -- --template react-ts
cd markdown-viewer
npm install react-router-dom react-markdown remark-gfm
```

**成果物**:
- TypeScript + React + Viteの環境構築
- 必要なパッケージのインストール完了

### Phase 2: コア機能の実装

**実行時間**: 約10分

**実装内容**:
1. MarkdownViewerコンポーネント
2. React Routerによるルーティング
3. サイドバーナビゲーション
4. Markdownスタイリング

**生成されたファイル**:
- `src/components/MarkdownViewer.tsx`
- `src/components/MarkdownViewer.css`
- `src/App.tsx`
- `src/App.css`

### Phase 3: ダウンロード機能の追加

**実行時間**: 約8分

**実装内容**:
1. DownloadPageコンポーネント
2. ファイル一覧の動的表示
3. ダウンロード機能
4. レスポンシブデザイン

**生成されたファイル**:
- `public/dl/` ディレクトリ
- `src/components/DownloadPage.tsx`
- `src/components/DownloadPage.css`
- `public/dl/files.json`

### Phase 4: デプロイ

**実行時間**: 約5分

```bash
# AIが実行したコマンド
git init
git add .
git commit -m "Initial commit"
npx vercel login
npx vercel --prod
```

**成果物**:
- 本番環境URL: https://ojt-seven-liart.vercel.app

### Phase 5: リポジトリの公開

**実行時間**: 約3分

```bash
# AIが実行したコマンド
winget install --id GitHub.cli
gh auth login
gh repo edit Goodbye-Antwerpbule/ojt --visibility public
git push -u origin main
```

**成果物**:
- 公開リポジトリ: https://github.com/Goodbye-Antwerpbule/ojt

## メリットとデメリット

### メリット

#### 1. 開発速度の向上
- プロジェクトのセットアップが数分で完了
- 繰り返しタスクの自動化
- ボイラープレートコードの自動生成

#### 2. コード品質の向上
- ベストプラクティスの自動適用
- 一貫したコーディングスタイル
- TypeScriptによる型安全性

#### 3. 学習機会の増加
- 実装の過程を見ながら学習
- 新しい技術スタックへの迅速な対応
- 説明付きのコード生成

#### 4. ドキュメンテーション
- 自動的なコメント追加
- README、CLAUDE.mdの生成
- このような詳細なドキュメントの作成

### デメリット

#### 1. 依存性
- AIツールへの依存度が高まる
- オフライン作業が困難
- コスト（API利用料など）

#### 2. 理解の重要性
- 生成されたコードの理解が必要
- デバッグスキルは依然重要
- セキュリティリスクの確認が必要

#### 3. 制限事項
- 複雑なビジネスロジックは人間の判断が必要
- プロジェクト固有の要件への対応
- パフォーマンスチューニング

## ベストプラクティス

### 1. 明確な指示を出す

**良い例**:
```
"TypeScriptとReactでMarkdownビューアーを作成。
react-markdown と remark-gfm を使用し、
public/docs/のMarkdownファイルを表示できるようにして"
```

**悪い例**:
```
"なんかいい感じのサイト作って"
```

### 2. 段階的な開発

一度にすべてを求めず、機能ごとに分けて実装：

1. 基本機能の実装
2. 追加機能の実装
3. スタイリング
4. デプロイ

### 3. 生成されたコードのレビュー

- セキュリティの確認
- パフォーマンスのチェック
- ベストプラクティスとの照合

### 4. バージョン管理

```bash
# 定期的なコミット
git add .
git commit -m "Add feature X"

# 重要なマイルストーンでのタグ付け
git tag -a v1.0 -m "First release"
```

### 5. ドキュメンテーション

- CLAUDE.mdでプロジェクトの指針を明記
- README.mdで使い方を説明
- コード内コメントで重要な部分を説明

### 6. テスト

AIが生成したコードもテストが必要：

```bash
# テストの実行
npm test

# ビルドの確認
npm run build
npm run preview
```
